Payment Gateway API

Tài liệu kỹ thuật để khởi tạo và truy vấn giao dịch thanh toán được thực hiện trên thiết bị máy POS.

Phiên bản 1.0 · Cập nhật 01/07/2026

01Tổng quan

Payment Gateway cung cấp hai API REST để Hệ thống Đối tác kết nối và điều khiển giao dịch thanh toán trên máy POS:

  • Khởi tạo Thanh toán — gửi yêu cầu tạo giao dịch mới trên máy POS (thẻ, QR, hoặc thành viên).
  • Truy vấn Giao dịch — tra cứu chi tiết một giao dịch đã thực hiện trên máy POS, theo mã giao dịch.

02Môi trường

Mọi endpoint bên dưới được gọi theo base URL tương ứng với môi trường:

Dev / SIT
https://sit-pos-gateway.onefin.vn
Production
https://pos-gateway.onefin.vn

03Chi tiết API

POST {base_url}/api/RequestPayment

Khởi tạo Thanh toán

Khởi tạo một yêu cầu thanh toán mới trên máy POS. Header Content-Type: application/json.

Request body

TrườngKiểuBắt buộcMô tả
AmountlongCóSố tiền giao dịch (VND, không có dấu thập phân).
SerialstringCóSố seri thiết bị/terminal thực hiện giao dịch.
RequestIdstringCóMã yêu cầu duy nhất do đối tác sinh ra, dùng để đối soát và chống trùng lặp.
InvoiceNumberstringKhôngSố hóa đơn của giao dịch.
Typeint (enum)CóHình thức thanh toán — xem TransactionFormType bên dưới.

Enum · TransactionFormType

Giá trịTênMô tả
1CardThanh toán bằng thẻ.
2QRThanh toán bằng mã QR.
3MemberThanh toán bằng tài khoản thành viên.

Ví dụ request

{
  "Amount": 100000,
  "Serial": "047547",
  "RequestId": "UNIQUE_REF_20260701_001",
  "InvoiceNumber": "INVOICE_20260701_001",
  "Type": 1
}

Response

Trả về đối tượng response — xem cấu trúc chi tiết tại mục 3.3.

GET {base_url}/api/QueryTransaction/{id}

Truy vấn Giao dịch

Tra cứu thông tin chi tiết của một giao dịch đã thực hiện trên máy POS.

Path parameter

TrườngKiểuBắt buộcMô tả
idstringCóMã giao dịch (transId) hoặc mã yêu cầu (RequestId) của giao dịch cần truy vấn.

Ví dụ request

GET https://sit-pos-gateway.onefin.vn/api/QueryTransaction/TRANS_20260701_00123

Response

Trả về response nếu tìm thấy giao dịch. Nếu không hợp lệ hoặc không tìm thấy, API trả về đối tượng với status.code = "12", các trường data, header, requestData đều null.

3.3Cấu trúc dữ liệu response

Cả hai API trên đều trả về (hoặc bọc) cấu trúc dữ liệu chung sau.

04Webhook — Nhận kết quả giao dịch

Khi giao dịch thanh toán trên máy POS hoàn tất (thành công hoặc thất bại), hệ thống sẽ chủ động gửi POST dữ liệu kết quả giao dịch đến Webhook URL mà đối tác đã đăng ký, giúp đối tác cập nhật trạng thái đơn hàng theo thời gian thực mà không cần chủ động gọi lại API Truy vấn Giao dịch.

Đối tác cần cung cấp Webhook URL khi đăng ký tích hợp. Endpoint này cần hỗ trợ phương thức POST với header Content-Type: application/json.

Payload gửi đến Webhook

Cấu trúc dữ liệu gửi đến Webhook tương tự cấu trúc response đã mô tả tại mục 3.3, gồm các trường requestId, requestData, header, data và status.

  "requestId": "UNIQUE_REF_20260701_001",
  "requestData": {
    "type": "card",
    "action": "sale",
    "merchant_request_data": "{...}"
  },
  "header": {
    "transId": "TRANS_20260701_00123",
    "provider": "NAPAS",
    "transType": "Sale",
    "terminalId": "047547",
    "merchantId": "50A069608",
    "merchantTransId": "UNIQUE_REF_20260701_001",
    "transmitsDateTime": "2026-07-01T10:15:30"
  },
  "data": {
    "status": "00",
    "totalAmount": "100000",
    "currency": "704",
    "cardBrand": "VISA",
    "cardHolder": "NGUYEN VAN A",
    "cardNumber": "4xxxxxxxxxxx1234",
    "cardMode": 2,
    "approveCode": "123456",
    "isoResponseCode": "00",
    "refNo": "REF00123",

  },

    "code": "00",
    "message": "Giao dịch thành công"

Phản hồi từ đối tác

Sau khi nhận và xử lý thành công dữ liệu, endpoint Webhook của đối tác cần trả về mã HTTP 200 để xác nhận đã nhận. Đối tác cũng có thể tự đối soát lại bất kỳ lúc nào bằng API Truy vấn Giao dịch với transId hoặc RequestId tương ứng.

05Mã lỗi

Trường status.code trong response xác định kết quả xử lý:

CodeÝ nghĩa
00Giao dịch thành công.
khác 00Giao dịch thất bại hoặc có lỗi. Xem nội dung status.message đi kèm để biết chi tiết nguyên nhân.

Ví dụ: 12 → "12 - Giao dịch không hợp lệ"

06Ví dụ tích hợp (cURL)

Ví dụ dùng domain môi trường Dev/SIT. Khi chuyển sang Production, thay bằng https://pos-gateway.onefin.vn.

Khởi tạo thanh toán

curl -X POST "https://sit-pos-gateway.onefin.vn/api/RequestPayment" \
  -H "Content-Type: application/json" \
  -d '{
    "Amount": 100000,
    "Serial": "047547",
    "RequestId": "UNIQUE_REF_20260701_001",
    "Type": 1
  }'

Truy vấn giao dịch

curl -X GET "https://sit-pos-gateway.onefin.vn/api/QueryTransaction/TRANS_20260701_00123"
