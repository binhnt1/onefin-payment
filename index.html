Tài liệu Kỹ thuật Tích hợp Thanh toán App-to-App

Phiên bản: 1.0 | Cập nhật: 11/08/2025

1. Tổng quan

Tài liệu này mô tả quy trình tích hợp kỹ thuật giữa Ứng dụng Đối tác (Merchant App) và Ứng dụng Thanh toán OneFin (Payment App) trên nền tảng Android. Mục tiêu là cho phép Ứng dụng Đối tác có thể khởi tạo các yêu cầu thanh toán và truy vấn thông tin giao dịch một cách an toàn và liền mạch thông qua ứng dụng OneFin.

Luồng tích hợp bao gồm hai nghiệp vụ chính:

  • Thực hiện Giao dịch: Khởi chạy giao diện thanh toán của OneFin để xử lý các nghiệp vụ như Bán hàng (Sale), Hoàn tiền (Refund), v.v.
  • Truy vấn Giao dịch: Truy vấn thông tin chi tiết của một giao dịch đã hoàn thành dựa trên mã hóa đơn (Bill Number).

2. Luồng Tích hợp Kỹ thuật

2.1. Thực hiện Giao dịch (Launch for Payment)

Để khởi tạo một giao dịch, Ứng dụng Đối tác cần tạo và gửi một `Intent` đến ứng dụng OneFin.

Cấu trúc Intent

  • Action: com.onefin.paymentapp.MerchantAction
  • Data URI: paymentapp://com.onefin.merchantapp
  • Method: startActivityForResult()

Dữ liệu đính kèm (Extras)

Tên KeyKiểu dữ liệuBắt buộcMô tả
`type`StringCóLoại phương thức thanh toán. Ví dụ: "card", "member", "qr".
`action`IntCóMã hành động cần thực hiện. Ví dụ: 1 (Sale), 2 (Refund).
`merchant_request_data`StringCóChuỗi JSON chứa thông tin chi tiết của yêu cầu.

Cấu trúc `merchant_request_data` (JSON String)

{
  "ccy": "704",
  "reference_id": "UNIQUE_REF_123",
  "message": "Thanh toan cuoc taxi",
  "bill_number": "TOT1234567",
  "amount": 100000,
  "tip": 0,
  "mid": "50A069608",
  "tid": "047547",
  "is_enter_pin": true,
  "additional_data": {
    "trip_id": "TRIP_XYZ",
    "driver_id": "047547",
    "driver_name": "Ngô Thanh Bình"
  }
}

2.2. Truy vấn Lịch sử Giao dịch (Query Transaction)

Ứng dụng Đối tác có thể truy vấn thông tin của một giao dịch đã thực hiện thông qua `ContentProvider` của ứng dụng OneFin.

Cấu trúc URI

  • Định dạng: content://authority/path/id
  • Authority: com.onefin.memberapp.transaction.provider
  • Path: transID
  • ID: Giá trị `bill_number` của giao dịch.

Dữ liệu trả về (Cursor)

Tên CộtKiểu dữ liệuMô tả
`action`IntMã hành động của giao dịch. Ví dụ: 6.
`member_response_data`StringChuỗi JSON chứa chi tiết giao dịch.

3. Yêu cầu về Quyền và Khai báo (Manifest)

Để đảm bảo ứng dụng có thể tương tác với ứng dụng OneFin trên Android 11+, cần khai báo thẻ `` trong file `AndroidManifest.xml`.

<manifest ...>
    ...
    <queries>
        <!-- Cho phép truy vấn ContentProvider -->
        <provider 
            android:authorities="com.onefin.memberapp.transaction.provider" />

        <!-- Cho phép gửi Intent với Action tùy chỉnh -->
        <intent>
            <action android:name="com.onefin.paymentapp.MerchantAction" />
        </intent>
    </queries>
</manifest>

4. Mã nguồn Tham khảo (Reference Code)

Dưới đây là đoạn mã Kotlin đầy đủ cho `MainActivity.kt` trong một dự án Flutter, thể hiện cách triển khai cả hai nghiệp vụ.

package com.example.mailinh_app

import android.app.Activity
import android.content.Intent
import android.database.Cursor
import android.net.Uri
import io.flutter.embedding.android.FlutterActivity
import io.flutter.embedding.engine.FlutterEngine
import io.flutter.plugin.common.MethodChannel

class MainActivity : FlutterActivity() {
    private val REQUEST_CODE = 1001
    private val CHANNEL = "com.mailinh/payment_channel"
    private var resultCallback: MethodChannel.Result? = null

    private fun query(billNumber: String): Map<String, Any>? {
        val authority = "com.onefin.memberapp.transaction.provider"
        val path = "transID"
        val uri = Uri.parse("content://$authority/$path/$billNumber")

        val cursor: Cursor? = contentResolver.query(uri, null, null, null, null)

        cursor?.use {
            if (it.moveToFirst()) {
                val actionIndex = it.getColumnIndex("action")
                val dataIndex = it.getColumnIndex("member_response_data")

                val action = if (actionIndex != -1) it.getInt(actionIndex) else -1
                val responseData = if (dataIndex != -1) it.getString(dataIndex) else ""

                return mapOf(
                    "action" to action,
                    "member_response_data" to responseData
                )
            }
        }
        return null
    }

    override fun configureFlutterEngine(flutterEngine: FlutterEngine) {
        super.configureFlutterEngine(flutterEngine)
        MethodChannel(flutterEngine.dartExecutor.binaryMessenger, CHANNEL)
            .setMethodCallHandler { call, result ->
                when (call.method) {
                    "launchPaymentApp" -> {
                        resultCallback = result
                        val type = call.argument<String>("type")
                        val action = call.argument<Int>("action")
                        val merchantRequestData = call.argument<String>("merchant_request_data")
                        try {
                            val intent = Intent("com.onefin.paymentapp.MerchantAction")
                            intent.data = Uri.parse("paymentapp://com.onefin.merchantapp")
                            intent.addCategory(Intent.CATEGORY_DEFAULT)
                            intent.putExtra("type", type)
                            intent.putExtra("action", action)
                            intent.putExtra("merchant_request_data", merchantRequestData)
                            startActivityForResult(intent, REQUEST_CODE)
                        } catch (e: Exception) {
                            result.error("INTENT_ERROR", "Could not launch payment app", e.localizedMessage)
                            resultCallback = null
                        }
                    }
                    "queryTransaction" -> {
                        try {
                            val billNumber = call.argument<String>("bill_number")
                            if (billNumber == null) {
                                result.error("INVALID_ARGUMENT", "bill_number is required.", null)
                                return@setMethodCallHandler
                            }
                            val transactionData = query(billNumber)
                            result.success(transactionData)
                        } catch (e: Exception) {
                            result.error("QUERY_FAILED", "Failed to query transaction.", e.message)
                        }
                    }
                    else -> result.notImplemented()
                }
            }
    }

    override fun onActivityResult(requestCode: Int, resultCode: Int, data: Intent?) {
        super.onActivityResult(requestCode, resultCode, data)
        if (requestCode == REQUEST_CODE) {
            resultCallback?.let {
                if (resultCode == Activity.RESULT_OK && data != null) {
                    val response = data.getStringExtra("payment_response_data") ?: "No response"
                    it.success(response)
                } else {
                    it.error("CANCELLED", "User cancelled or no data", null)
                }
                resultCallback = null
            }
        }
    }
}
